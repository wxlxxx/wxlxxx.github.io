1.1 什么是单页应用

只有一张Web页面的应用,单页面跳转仅刷新局部资源, 是一种从Web服务器加载的富客户端, 公共资源(js、css等)仅需加载一次

1.2 单页应用的优点

1. 操作体验流畅（资源加载完运行在本地，与服务端基本只有数据的交互）

2. 组件化的维护 (易于拓展与移植)

3. 良好的前后端分离 (前端负视图、路由、一定的业务逻辑。。)

1.2 单页应用的缺点

1. 首次加载大量资源 (要注重模块化开发和架构设计)

2. 搜索引擎不友好

3. 开发难度相对较高

2.1 开发单页应用注意事项

1. 选择恰当的开发框架

2. 代码合并与分割、加载策略

3. 路由与状态的管理

4. 缓存与本地存储

3.1 语言/框架的选择

原生html css js开发效率低、代码臃肿不优雅

3.2 javascript框架的选取的原则

1.高效的渲染能力

2.响应式组件

3.强大的开发者社区
