1.1 什么是单页应用

demo1

demo2

只有一张Web页面的应用,单页面跳转仅刷新局部资源, 是一种从Web服务器加载的富客户端, 公共资源(js、css等)仅需加载一次

1.2 单页应用的优点

1. 操作体验流畅（资源加载完运行在本地，与服务端基本只有数据的交互）

2. 组件化的维护 (后面介绍维护组件相比维护页面的好处)。

3. 良好的前后端分离 (前端负责视图渲染、路由、一定的业务逻辑。。)

1.3 维护组件相比维护页面的好处

1.好的组件具有也必须具有高内聚低耦合的特点

2.避免牵一发而动全身

3.避免同样的逻辑在多个地方重复书写

4.更好的团队开发分工

1.4 单页应用的缺点

1. 首次加载大量资源 (要注重模块化开发和架构设计)

2. 搜索引擎不友好 (解决方法)

3. 开发难度相对较高

2.1 开发单页应用注意事项

2. 代码合并与分割、加载策略

6.2 状态管理

Web 单页应用就是一个状态机，视图与状态是一一对应的。

以redux为例说明

1.所有的状态，保存在一个对象里面。

2.Redux 规定， 一个 State 对应一个 View。只要 State 相同，View 就相同。你知道 State，就知道 View 是什么样，反之亦然。

3.由Store、Action、Reducer三部分组成
